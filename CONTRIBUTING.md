# Contributing to WorthIt

Thank you for your interest in contributing to WorthIt! We welcome contributions from everyone.

## Code of Conduct

Be respectful, inclusive, and constructive in all interactions.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/worthit.git`
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Follow the development setup in README.md

## Development Guidelines

### Code Style
- Use TypeScript for all new code
- Follow ESLint and Prettier configurations
- Use meaningful variable and function names
- Add JSDoc comments for complex functions

### Commit Messages
- Use conventional commits: `feat:`, `fix:`, `docs:`, `style:`, `refactor:`, `test:`, `chore:`
- Example: `feat: add purchase recommendation scoring algorithm`
- Keep messages clear and descriptive

### Pull Requests
1. Keep PRs focused on a single feature or fix
2. Write clear PR descriptions
3. Reference related issues
4. Ensure all tests pass
5. Request review from maintainers

## Testing

```bash
# Run tests
npm test

# Run with coverage
npm run test:coverage

# Run linting
npm run lint
```

## Documentation

- Update relevant documentation files
- Add comments for complex logic
- Keep README.md up to date
- Document new API endpoints

## Reporting Bugs

1. Check if the issue already exists
2. Provide clear reproduction steps
3. Include system information
4. Attach screenshots/logs if relevant
5. Use bug report template

## Feature Requests

1. Describe the use case
2. Explain the expected behavior
3. Include mockups if helpful
4. Consider performance implications

## Questions?

Create a discussion or reach out to the maintainers.

Thank you for contributing! 🚀
